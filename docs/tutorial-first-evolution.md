# Your First Evolution

This tutorial walks you through running KRAIT and triggering your first skill evolution.

## Prerequisites

- Docker and Docker Compose
- An OpenRouter API key ([openrouter.ai/keys](https://openrouter.ai/keys))

## Quickstart (5 minutes)

### 1. Clone and Start

```bash
git clone https://github.com/postrv/krait.git
cd krait

# Set your API key
export OPENROUTER_API_KEY=sk-or-...

# Start KRAIT
docker compose -f docker-compose.quickstart.yml up
```

Watch the logs for your admin and API tokens:

```
=== KRAIT Quickstart ===

Admin dashboard: http://localhost:4000/admin/login
Admin token:     <auto-generated>
API token:       <auto-generated>

========================
```

### 2. Log In

Open [http://localhost:4000/admin/login](http://localhost:4000/admin/login) and enter the admin token from the console output.

### 3. View the Feed

After logging in, you'll see the Evolution Feed dashboard with 5 demo evolution events:

- **text_transform** — Text transformation utilities
- **json_tools** — JSON manipulation
- **math_utils** — Mathematical utilities
- **date_helper** — Date/time helpers
- **code_metrics** — Elixir code analysis (draft)

Click any event card to expand details: AST hash, test count, complexity metrics, and reasoning.

### 4. Trigger an Evolution

Use the "Trigger Evolution" form above the feed:

1. **Skill name**: Enter a snake_case name (e.g., `greeting`)
2. **Description**: Describe what the skill should do (e.g., "Generate greeting messages in different languages and tones")
3. Click **Evolve**

The system will:
1. Validate the skill name
2. Check the kill switch status
3. Acquire an evolution slot
4. Generate code via the LLM
5. Validate against all 7 KRAIT security rules (dual-engine)
6. Run tests
7. Record the result in the feed

> **Note**: In quickstart mode (KRAIT_DRY_RUN=true), the GitHub integration uses a dry-run client. No real PRs are created.

### 5. Explore the API

List skills:
```bash
curl -H "Authorization: Bearer <your-api-token>" \
  http://localhost:4000/api/skills
```

View the feed:
```bash
curl -H "Authorization: Bearer <your-api-token>" \
  http://localhost:4000/api/feed
```

## Full Setup (Production)

For production deployment with real GitHub integration and Docker sandbox:

### 1. Generate Secrets

```bash
./scripts/setup-prod-secrets.sh
```

This creates `.env.prod` with all required secrets.

### 2. Configure

Edit `.env.prod`:
- Set `OPENROUTER_API_KEY` to your real key
- Set `PHX_HOST` to your domain
- Optionally configure a GitHub App for real PR creation

### 3. Start Production Stack

```bash
docker compose -f docker-compose.prod.yml up -d
```

This includes:
- **PostgreSQL** with pgvector
- **DinD sandbox** with TLS, userns-remap, seccomp
- **KRAIT** application server

### 4. GitHub App (Optional)

To create real PRs during evolution:

1. Create a GitHub App at `https://github.com/settings/apps`
2. Grant permissions: Contents (read/write), Pull Requests (read/write)
3. Install on your target repository
4. Set environment variables:
   - `GITHUB_APP_ID`
   - `GITHUB_PRIVATE_KEY_PATH`
   - `GITHUB_INSTALLATION_ID`
   - Remove `KRAIT_DRY_RUN=true` from your config

## Understanding the Security Model

Every evolution goes through this pipeline:

```
LLM generates code
    │
    ▼
Quick Analyzer (Elixir AST)  ──── deny ──→ reject + record
    │ pass
    ▼
NIF Analyzer (Rust tree-sitter)  ──── deny ──→ reject + record
    │ pass
    ▼
Immutable path check  ──── match ──→ reject + record
    │ pass
    ▼
Docker sandbox execution
    │
    ▼
Test suite runs
    │
    ▼
Feed event recorded (draft or merged)
```

The 7 KRAIT rules checked by both analyzers:

| Rule | What it blocks |
|------|---------------|
| KRAIT-001 | Code evaluation (`Code.eval_string`, `EEx.eval_string`) |
| KRAIT-002 | Shell access (`System.cmd`, `:os.cmd`, `Port.open`) |
| KRAIT-003 | Credential paths (`~/.ssh`, `~/.aws`) |
| KRAIT-004 | Network exfiltration (raw HTTP clients) |
| KRAIT-005 | Hot code loading (`Code.load_file`) |
| KRAIT-006 | Immutable path targeting (`native/`, `.krait-immutable`) |
| KRAIT-007 | Self-modification (evolution/analyzer modules) |

## Troubleshooting

### "Kill switch is active"

The kill switch auto-trips after consecutive failures. Resume:
```bash
curl -X POST -H "Authorization: Bearer <admin-token>" \
  http://localhost:4000/api/kill-switch/resume
```

### "Too many concurrent evolutions"

KRAIT limits concurrent evolutions (default: 3). Wait for in-progress evolutions to complete.

### Database connection errors

Ensure PostgreSQL is healthy:
```bash
docker compose -f docker-compose.quickstart.yml ps
```

### Container won't start

Check logs:
```bash
docker compose -f docker-compose.quickstart.yml logs krait
```

Common issues:
- Missing `OPENROUTER_API_KEY`
- Port 4000 already in use (change with `KRAIT_PORT=4001`)
